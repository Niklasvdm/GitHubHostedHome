So I needed a website

I am not a huge fan of writing front-ends, spinning up WordPress instances, or generally spending hours tinkering with front-end frameworks.

While talking to one of my good friends (shoutout to Ruben), he mentioned he had his whole site running via GitHub Pages. I could not believe it, so I added “ship my own GitHub-hosted site” to the project list.

Getting started with GitHub Pages

As with many weekend projects, I kicked things off by cloning something that already looked great: rubenvanlaer, source code: ruben-vl/personal-website.

Using that as a base, I kept the clean structure but swapped in my own sections, content, and styling tweaks so the site felt like mine. niklasvdm/GithubHostedHome

Adding a Custom domain

Pointing my own domain to the project was easier than expected by following GitHub’s custom domain guide. It walks through the DNS records, the `CNAME` file, and HTTPS verification in a few concise steps.

I personally use GoDaddy, which required that I deleted some automatic CNAME records and making sure the only CNAME record configured, is pointing to the GitHub pages as is specified within the provided documentation. I ignored the AAA records for now as I'm not too worried about IPv6 due to this being managed by Microsoft in this case. I only added the A records and I actually am looking into cleaning up my other A rules in favour of AAA rules for my other subdomains.

I'm happy I have already struggled with this in the past, as I now have sufficient documentation about DNS records that I can easily find the information I need.

Adding a landing page on github

The finishing touch was a simple landing page for my GitHub profile. I loosely followed this quick Instagram walkthrough and relied on ChatGPT to provide the boilerplate content, changing some stuff to align with what I'm working on and not providing too much info on what I am not too proud of...

You simply make a new public repo that has the same name as your GitHub name. In this way, the README will display as the information presented on your profile.

So I simply had to add ´Niklasvdm/Niklasvdm´, add an automatic README.md and I was ready to get started adding the content!

The result is now displayed my GitHub landing page. Ignore the fact that I don't seem to push much to GitHub (I've never truly cared about having contributions, otherwise I'd have automated it by now), and simply copy-paste some emoji-filled, chaptgpt-generated content into your own README for fast & quick boilerplating.

If I'm ever looking for a Developer job again (god I hope I won't have to but chances are not in my young-person favour), I might change it, but at least I won't have to look up how to set it up again and I can simply edit the content within the repo's README.

Takeaways

  • Cloning a proven template let me focus on content instead of fiddling with layout.
  • GitHub Pages plus DNS tweaks is all you need to serve a custom domain without running infrastructure.
  • Using ChatGPT to generate boilerplate HTML/CSS snippets really is a lifesavor. It made me not dread making this project since I can rely on it to deal with the unpleasant formatting.
  • A polished GitHub profile page takes less than an hour when you start with good references.
